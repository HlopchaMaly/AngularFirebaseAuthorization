Что происходит?

Данный проект изначально был задуман с целью получения/улучшения практических навыков использования платформ Angular и Firebase. Конечной целью проекта должен стать менеджер задач с оповещением на электронную почту о приближении срока выполнения задания, а также присвоением «звания» пользователю в зависимости от соотношения выполненных и проваленных заданий.

На данном этапе написания проекта реализованы функции регистрации нового пользователя, удаления аккаунта, авторизации уже существующего и изменения личных данных пользователя. Функционал непосредственно «менеджера задач» пока не реализован.

Схематическое расположение компонентов:

Технологии, сервисы, средства

В данном разделе я попытался обозначить сервисы платформы Firebase, средства фреймворка Angular, и другие инструменты, исползуемые в проекте, с указанием места их применения.

Firebase

  • Firebase Authentication - используется для создания аккаунта пользователя, удаления аккаунта, входа пользователя в уже существующую учетную запись, выхода из аккаунта, а также изменения личных данных зарегистрированного пользователя. Применяется в проекте в сервисах auth.service.ts и user.service.ts.
  • Firebase Realtime Database - NoSQL облачная база данных. На данном этапе написания приложения используется для хранения пользователя и его данных. Расширяет функционал Firebase Authentication, путем создания записи в БД, с данными аналогичными, содержащимся в firebase.auth().currentUser с добавлением дополнительных полей в запись. Применяется в user.service.ts.
  • Firebase Cloud Storage - облачное хранилище, применяемое для хранения файлов. В конкретном случае используется для хранения какртинок-аватаров пользователя, загруженных с локальной ЭВМ. Реализовано в user.service.ts.
  • Firebase Cloud Functions - облачные функции. В проекте присутсвует функция для удаления аккаунта вновь зарегистрированного пользователя, если email, указанный при регистрации, не был подтвержден в течение часа. Пользователь удаляется как из Firebase Authentication, так и из Firebase RTDB. Функция реализована в корне проекта в каталоге functions.

Angular

Привязка данных:

  • Иннтерполяция - user-info.component.html например, строки 15, 25.
  • Односторонняя привязка свойства элемента DOM к значению компонента. Например, строка 47 change-user-data.component.html.
  • Привязка метода компонента к событию в DOM - change-user-data.component.html строка 47.
  • Class binding - change-user-data.component.html строка 48.
  • Использование шаблонных переменных, ViewChild - change-user-data.component для фокусировки на соответсвующем инпуте в зависимости от параметра строки запроса.

Взаимодействие между компонентами:

  • Передача данных в дочерний компонент - app.component.html строка 11.
  • Взаимодействие между не родственными компонентами посредством сервиса. Реализовано для сворачивания-разворачивания навбара компонента ui-home.component при клике по "гамбургеру" в хеддере header.component посредством сервиса common.service.ts. Аналогичным образом, при помощи сервиса modal.service.ts, сообщаются компоненты user-info.component и app.component с целью вызова и отображения тултипа - tooltip.component.

Директивы:

  • Атрибутивные директивы. ngStyle - user-info.component.html строка 6.
  • Структурные директивы. ngIf - например, user-info.component.html строка 6 и ngFor - slider.component.html.

Сервисы:

В проекте применяется ряд сервисов, для выполнения различных задач - отображения модальных окон и тултипов, предотвращения несанкционированного перехода по маршрутам, авторизации и изменения данных пользователя. Лежат в папке services .

Формы:

Работа с формами осуществляется с использованием подхода Reactive Forms. Примеры использования - change-user-data.component, а также в компонентах-модальных окнах app/components/modal .

Маршрутизация:

  • В проекте применена техника Lazy Loading. Основной модуль с маршрутами - app.routing.module, дочерние - ui-home.module, change-user-data.module, tasks-home.module.
  • Параметр строки запроса - user-info.component.ts строка 69.
  • Guard - в проекте используется для предотвращения перехода на главную страницу UI (ui-home.component) неавторизованного пользователя, а также автоматического выхода из аккаунта при уходе с данной страницы. Для выполнения вышеуказанных действий создан сервис guard.service.ts.

Pipes:

В проекте применяется AsyncPipe для отображения данных пользователя в шаблоне - user-info.component.html например, строка 25.

Анимация:

Организована с использованием Angular animations. Применяется в компоненте ui-home.component для сворачивания-разворачивания навбара (на определенных разрешениях экрана).

Rxjs:

Из библиотеки rxjs в проекте применены классы Subject и Observable в контексте сервисов tooltip.service.ts и common.service.ts. Также для "отписки" от сервисов в соответствии с жизненным циклом компонента, везде, где это необходимо, применяется класс Subscription.

Other
